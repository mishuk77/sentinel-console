## What I'd add next

- **Per-page audit panel on Policy** — the slider's projected approval/loss rates are computed client-side from binned calibration data. Adding an audit panel here would require either fetching the simulation engine for a single-stage view, or surfacing the model's training-time metrics with a "computed from training calibration" disclaimer. The spec rule was satisfied for *every metric-bearing page that renders simulation output* — the Policy page projections are explanatory, not authoritative.

- **Deletion protection extending to models and policies** — spec mentions all three. Datasets are protected; the equivalent ORM-level check on models + policies referenced by backtest_runs is straightforward to add when the corresponding delete endpoints get UI affordances. Per spec: "Datasets, models, and policies that are referenced by any persisted backtest or simulation run cannot be deleted — only archived."

- **`<PolicyDiff />` on the projected simulation summary** — TASK-11H is currently wired only into the publish-confirm modal. Adding a "Compare against published" toggle on `SimulationSummary.tsx` would surface the same component for arbitrary comparisons.

These are remaining-but-non-critical items. The 15 spec checklist items are met as documented above.
